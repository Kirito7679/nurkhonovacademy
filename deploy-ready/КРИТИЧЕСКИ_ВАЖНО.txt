ШАГ 1: ПРОВЕРЬТЕ СОДЕРЖИМОЕ ПАПКИ assets/ НА СЕРВЕРЕ
─────────────────────────────────────────────────────

В cPanel → File Manager:
1. Откройте папку academy/assets/
2. Должно быть РОВНО 3 файла:
   ✅ index-COOPZO17.js
   ✅ index-LLn-KSah.css
   ✅ Preview-J4hSyiZS.js

Если файлов меньше или они с другими именами:
→ УДАЛИТЕ все файлы из assets/
→ ЗАГРУЗИТЕ все 3 файла из deploy-ready/assets/

ШАГ 2: ПРОВЕРЬТЕ API
─────────────────────

Откройте в браузере:
https://api.academy.dilmurodnurkhonov.uz/api/health

Должно вернуть: {"success":true,"message":"Server is running"}

Если не работает:
→ Проверьте Railway Dashboard
→ Проверьте логи Railway

ШАГ 3: ОЧИСТИТЕ КЭШ ПОЛНОСТЬЮ
──────────────────────────────

1. Откройте DevTools (F12)
2. Правый клик на кнопке обновления
3. Выберите "Очистить кэш и жесткая перезагрузка"

Или:
1. Chrome: Settings → Privacy → Clear browsing data
2. Выберите "Cached images and files"
3. Очистите

ШАГ 4: ПРОВЕРЬТЕ В РЕЖИМЕ ИНКОГНИТО
────────────────────────────────────

Откройте сайт в режиме инкогнито:
- Там кэш не используется
- Если работает в инкогнито → проблема в кэше
- Если не работает → проблема в файлах или API

║                    ПРОВЕРКА В БРАУЗЕРЕ:                         ║
╚═══════════════════════════════════════════════════════════════╝

1. Откройте DevTools (F12) → Network
2. Обновите страницу
3. Проверьте, какие файлы загружаются:
   - ✅ Должен загружаться: index-COOPZO17.js
   - ❌ НЕ должен загружаться: index-FK4ubfmL.js (старый)

4. Проверьте Console на ошибки:
   - Если видите ошибки API → проблема с Railway
   - Если видите "t is not defined" → старые файлы
   - Если видите 404 для файлов → файлы не загружены
