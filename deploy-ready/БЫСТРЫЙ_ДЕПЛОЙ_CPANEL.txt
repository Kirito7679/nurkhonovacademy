  БЫСТРАЯ ИНСТРУКЦИЯ ПО ЗАГРУЗКЕ НА CPANEL
═══════════════════════════════════════════════════════════════

📦 ШАГ 1: ПОДГОТОВКА
───────────────────────────────────────────────────────────────
1. Создайте резервную копию текущего сайта (если есть)
2. Убедитесь, что у вас есть доступ к cPanel

📤 ШАГ 2: ЗАГРУЗКА ФАЙЛОВ
───────────────────────────────────────────────────────────────

ВАРИАНТ А: Через File Manager в cPanel
───────────────────────────────────────────────────────────────
1. Войдите в cPanel
2. Откройте "File Manager"
3. Перейдите в папку public_html (или нужную подпапку)
4. Удалите старые файлы (index.html, assets/, .htaccess)
5. Загрузите ВСЕ файлы из папки deploy-ready:
   ✓ index.html
   ✓ .htaccess
   ✓ assets/ (вся папка со всеми файлами)

ВАРИАНТ Б: Через FTP
───────────────────────────────────────────────────────────────
1. Откройте FTP-клиент (FileZilla, WinSCP и т.д.)
2. Подключитесь к серверу
3. Перейдите в public_html
4. Загрузите все файлы из deploy-ready

⚙️ ШАГ 3: ПРОВЕРКА .htaccess
───────────────────────────────────────────────────────────────
1. Убедитесь, что файл .htaccess загружен
2. В cPanel проверьте, что включен mod_rewrite
   (Apache Modules → mod_rewrite должен быть включен)

✅ ШАГ 4: ПРОВЕРКА
───────────────────────────────────────────────────────────────
1. Откройте сайт в браузере
2. Очистите кеш браузера (Ctrl+Shift+R)
3. Проверьте работу сайта

🐛 ЕСЛИ ЧТО-ТО НЕ РАБОТАЕТ
───────────────────────────────────────────────────────────────

Проблема: Страницы показывают 404
Решение:
  → Проверьте, что .htaccess загружен
  → Убедитесь, что mod_rewrite включен
  → Проверьте права доступа (644 для файлов, 755 для папок)

Проблема: CSS/JS не загружаются
Решение:
  → Проверьте, что папка assets/ загружена полностью
  → Проверьте пути в index.html
  → Очистите кеш браузера

Проблема: Показывается старая версия
Решение:
  → Очистите кеш браузера (Ctrl+Shift+R)
  → Очистите кеш CDN (если используется)
  → Проверьте настройки кеширования в cPanel
